# 职场汇报技能（work-report）

> 基于 Claude Skills 框架的职场汇报材料撰写与优化工具，覆盖工作进展汇报、方案汇报、述职报告/年度总结等场景。

## 功能概览

| 汇报类型 | 适用场景 | 核心结构 |
|---------|---------|---------|
| 一件事-工作进展 | 周报、月报、项目进度更新 | 结论先行 → 亮点 → 问题 → 下一步 |
| 一件事-方案汇报 | 立项申请、资源申请、方案评审 | 目标 → 为什么要干 → 需要什么资源 |
| 多件事汇报 | 述职报告、年度总结、综合说明 | 全景总览 → 统一极简结构 × N |

## 三种使用方式

1. **写新材料** — 用户提供主题/素材，按框架从零生成
2. **优化现有材料** — 用户提供草稿，识别问题并按框架改写
3. **写提纲** — 只输出结构框架，不填充内容

## 三种输出格式

1. **PPT 大纲** — 每页含标题+要点+备注，可直接用于制作PPT
2. **docx 文档** — 完整Word文档，可直接使用
3. **docx 模板** — 填空版模板，用户自行填入内容

## 多件事汇报的四大特殊规则

| 规则 | 说明 |
|------|------|
| **开场必给全景总览** | 30秒内说清共几个问题、哪些需拍板、哪些只是同步 |
| **统一极简结构 + 优先级排序** | 每个事项固定格式，有风险的放前面，已闭环的简短带过 |
| **主动隐藏细节** | 过程性细节全放附件，主汇报只讲核心信息 |
| **提前回应共性疑问** | 每个事项主动说清下一步安排和保障，不等领导问 |

## 安装方式

### 方式一：安装 .skill 文件（推荐）

下载 `work-report.skill` 文件，导入到支持 Claude Skills 的平台（如 Claude Code、Claude Desktop 等）。

### 方式二：手动部署

将 `work-report/` 目录复制到你的 skills 文件夹中，确保平台能够加载 `SKILL.md` 文件。

## 目录结构

```
work-report/
├── SKILL.md                          # 主技能文件：触发条件 + 工作流 + 速查表
├── references/
│   ├── report-frameworks.md          # 汇报框架详解（含完整示例）
│   └── output-formats.md             # 输出格式规范
└── assets/
    ├── work-progress-template.docx   # 工作进展汇报模板
    ├── proposal-template.docx        # 方案汇报模板
    └── multi-item-template.docx      # 多件事汇报模板
```

## 触发示例

以下是用户可能会说的触发语句：

- "帮我写一篇本周工作周报"
- "把这个项目进度整理成汇报材料"
- "帮我优化这份述职报告"
- "给我写一份方案汇报的 PPT 大纲"
- "给我一份工作进展汇报的 Word 模板"
- "帮我把这些内容组织成汇报逻辑"

## 依赖说明

- **PPT 大纲输出**：纯文本，无依赖
- **docx 模板输出**：直接读取模板文件，无依赖
- **docx 文档输出**：
  - 首选：若平台已安装 [document-skills](https://github.com/anthropics/skills/tree/main/document-skills)，调用其 docx 子技能生成实际 .docx 文件
  - 降级：直接输出 Markdown 格式文档，用户可复制到 Word
