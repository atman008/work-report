## PPT 大纲输出规范

### 适用场景

用户要求"写一份 PPT 大纲""给我汇报的 PPT 结构"或类似请求时使用。

### 每页结构

每页 PPT 包含三部分：

```
【第 X 页】页面标题
- 要点 1
- 要点 2
- 要点 3
（备注：讲稿提示，说明这页该怎么讲，时长建议）
```

**要点数量**：每页不超过 5 条，每条不超过 20 字。

**备注作用**：给汇报人看的讲稿提示，不是给听众看的正文。

### 标准页面类型

**1. 封面页**
```
【第 1 页】汇报主题
- 汇报人：XXX
- 日期：XXXX 年 XX 月 XX 日
（备注：开场 10 秒，直接报主题和身份）
```

**2. 目录/总览页（多件事汇报必做）**
```
【第 2 页】今天汇报 X 件事
- 事项 1：XXX（需决策）
- 事项 2：XXX（同步信息）
- 事项 3：XXX（已闭环）
（备注：30 秒全景总览，给听众安全感和时间预期）
```

**3. 内容页**
```
【第 X 页】事项标题
- 结论：一句话概括
- 核心进展：1-2 个关键数据
- 下一步：具体动作 + 时间
（备注：控制在 1-2 分钟，不展开细节）
```

**4. 总结页**
```
【第 X 页】核心结论
- 结论 1
- 结论 2
- 下一步重点
（备注：30 秒收束，强化记忆点）
```

**5. Q&A 页**
```
【第 X 页】Q & A
- 感谢聆听
- 开放提问
（备注：提前准备 3 个可能被问到的问题及回答要点）
```

### 页数控制

- 一件事汇报：5-8 页
- 多件事汇报：每事项 2-3 页 + 封面/总览/总结 3 页

## docx 文档输出规范

### 适用场景

用户要求"写一份汇报文档""生成 word 版"或类似请求时使用。

### 标题层级

| 层级 | 用途 | 格式 |
|------|------|------|
| 一级标题 | 汇报主题 | 黑体/加粗，18pt |
| 二级标题 | 各部分标题（结论/亮点/问题/下一步） | 黑体/加粗，14pt |
| 三级标题 | 子项或具体事项 | 黑体/加粗，12pt |
| 正文 | 内容描述 | 宋体/常规，12pt |

### 段落格式

**正文段落**：
- 行距 1.5 倍
- 段前段后 6pt
- 首行不缩进（职场汇报不用首行缩进）

**要点列表**：
- 使用项目符号（● 或 -）
- 每条独立成段
- 关键数据加粗

**表格**：
- 用于行动计划、资源需求等结构化信息
- 表头加底色区分
- 单元格内文字垂直居中

### 文档结构模板

**一件事-工作进展文档**：
```
一级标题：XX 工作进展汇报

一、当前结论
（1 段，核心结论）

二、主要亮点
（2-3 条，带数据）

三、存在问题
（1-2 条，带影响范围和解决思路）

四、下一步计划
（表格：行动项 | 负责人 | 完成时间 | 所需支持）
```

**一件事-方案汇报文档**：
```
一级标题：XX 方案汇报

一、目标
（SMART 描述）

二、背景与必要性
（背景 + 痛点 + 不做会怎样）

三、方案内容
（具体做法，分阶段）

四、资源需求
（表格：资源类型 | 具体内容 | 用途 | 必要性说明）

五、预期效果
（量化指标 + ROI 预估）
```

**多件事汇报文档**：
```
一级标题：XX 综合汇报

一、全景总览
（表格：序号 | 事项 | 当前状态 | 是否需要决策 | 信息类型）

二、事项一：XXX
（二级：结论 / 核心进展 / 下一步）

三、事项二：XXX
（同上结构）

...（逐项）

N. 总结与下一步重点
```

## docx 模板使用说明

### 模板定位

模板是**填空式起点**，不是最终文档。用户拿到模板后，按括号提示替换为自己的内容。

### 模板字体兼容性

模板使用宋体/黑体，若目标系统无该字体将自动回退为系统默认字体，不影响使用。

### 模板结构说明

每个模板包含：
- **固定区域**：标题、日期、汇报人等格式固定的位置
- **填空区域**：用【】标注，提示用户填写什么内容
- **示例区域**：用灰色文字给出填写示例，用户替换为实际内容

### 适用场景对照

| 用户需求 | 推荐模板 |
|---------|---------|
| "给我一份工作进展汇报模板" | work-progress-template.docx |
| "给我一份方案汇报模板" | proposal-template.docx |
| "给我一份述职/总结模板" | multi-item-template.docx |

### 生成方式

当用户要求"模板"时：
1. 读取对应模板文件
2. 保留【】填空标记和示例文字
3. 如有用户提供的信息，填入对应位置
4. 输出为 .docx 文件

## 与 document-skills 协作方式

当用户要求生成实际的 docx 或 pptx 文件时，调用 document-skills 子技能。

### 生成 docx 文档

```
Skill(document-skills/docx)
```

然后使用 docx-js 方式（JavaScript/TypeScript）生成文档：

- **必须阅读** `docx-js.md` 了解完整规范
- 使用 `Document`、`Paragraph`、`TextRun`、`Table` 等组件
- 通过 `Packer.toBuffer()` 输出

**关键规则**：
- 不用 `\n` 换行，用独立 `Paragraph`
- `PageBreak` 必须放在 `Paragraph` 内
- `ShadingType.CLEAR` 用于表格单元格底色
- 字号单位：half-points（12pt = 24）

### 生成 pptx 文件

```
Skill(document-skills/pptx)
```

**注意**：实际 pptx 生成较为复杂。如果用户只要求"PPT 大纲"，直接输出文本格式的大纲即可，不必须生成 .pptx 文件。只有当用户明确要求"生成 PPT 文件"时才调用 pptx 子技能。
